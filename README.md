RAG-Powered Website Chatbot (ClaySys Use Case)
A smart Retrieval-Augmented Generation (RAG) chatbot that answers user questions based on website content.
Built step-by-step over 7 days, this project demonstrates modern AI concepts, clean engineering, and a polished UI.

Tech Stack
Python
LangChain
FAISS (Vector Store)
HuggingFace Embeddings
Streamlit (Web UI)
Virtual Environment (venv)

Day-Wise Development Journey

Day 1 – Project Setup & Data Collection
Initialized GitHub repository
Created project structure
Collected website content
Stored raw text data for processing

Day 2 – Text Processing & Chunking
Cleaned website text
Split content into meaningful chunks
Prepared data for embeddings
Improved retrieval accuracy

Day 3 – Vector Store & RAG Pipeline
Generated embeddings using HuggingFace models
Stored vectors using FAISS
Built Retrieval-Augmented QA pipeline
Enabled question-answering from website data

Day 4 – Chatbot Logic
Created chatbot.py
Integrated retriever + LLM logic
Handled unknown questions safely
Enabled CLI-based chatbot interaction

Day 5 – Streamlit Web App
Built app.py using Streamlit
Connected chatbot backend to UI
Enabled real-time user queries
Displayed answers instantly

Day 6 – Error Handling & Stability
Added safe response handling
Improved reliability for missing answers
Cleaned dependency issues
Ensured smooth user experience

Day 7 – UI Polish & Final Touches 
Designed attractive, modern UI
Added colors, layout, and chat styling
Improved usability for interview demo
Finalized project for presentation
